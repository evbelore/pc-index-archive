# pc-index-archive

本仓库用于归档和发布多个独立的 HTML 页面。这些页面通常作为索引、导航或信息展示界面，不依赖特定后端服务，可直接在浏览器中打开。

## 项目简介

`pc-index-archive` 是一个静态 HTML 页面集合仓库，主要用途是将分散的、独立的页面文件集中存放，便于版本管理、历史回溯以及通过 GitHub Pages 等方式进行静态部署。

## 目录说明

```
pc-index-archive/
├── index.html        # 仓库总索引页（如有）
├── pages/            # 存放各独立 HTML 页面
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/           # 公共资源文件（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md         # 本文件
```

- `pages/`：核心目录，每个 HTML 文件为一个独立归档页面。
- `assets/`：存放页面引用的公共资源，按类型分目录组织。

## 页面归档说明

- 每个 HTML 页面均为独立单元，可脱机运行。
- 页面间无强耦合，不共享状态或数据。
- 归档时请保持文件命名清晰、语义化，便于检索。
- 如页面引用外部资源，建议将资源一并存放于 `assets/` 中，避免依赖失效。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，如有公共资源则放入 `assets/` 对应子目录。
- 更新页面：直接修改对应文件，建议附上简短的提交说明。
- 删除页面：移除文件前请确认无其他页面引用其资源。
- 索引页（如 `index.html`）可根据需要更新，以反映最新归档列表。

## 使用方式

1. 克隆仓库到本地。
2. 直接用浏览器打开 `pages/` 下的任意 HTML 文件即可查看。
3. 若启用 GitHub Pages，可设置 `pages/` 或根目录为发布源，在线访问归档页面。

## 许可

本仓库仅用于归档和展示，不构成任何形式的发布或授权。各页面内容版权归原作者所有。
